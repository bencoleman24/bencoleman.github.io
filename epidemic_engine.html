Overview

The Epidemic Engine is an integrated system designed to ingest, process, and analyze real-time health event data. Leveraging tools like Kafka, PostgreSQL, and PySpark, it enables anomaly detection through machine learning and provides live data visualization via a dynamic dashboard.

Key Components

  • Deployment: The pipeline is orchestrated using Podman Compose, ensuring reliability through health checks and automatic restarts. Services include Kafka, PostgreSQL, Consumer, Predictor, and a Visualization Dashboard.
  • Kafka: Streams health events continuously to the Consumer service.
  • Consumer Service: Processes Kafka events and stores them in the PostgreSQL database.
  • PostgreSQL Database: Hosts tables for health events and anomaly predictions.
  • Model Training: Uses PySpark to train an anomaly detection model on historical data, achieving >95% accuracy.
  • Predictor Service: Classifies recent events as anomalies using the trained model, updating every 10 minutes.
  • Visualization Dashboard: Displays live and historical health event data, including real-time map-based visualizations and interactive charts.

Features

  • End-to-end pipeline integrating real-time data ingestion, storage, analysis, and visualization.
  • Live dashboard updated every 10 seconds with new streamed data.
  • High-accuracy anomaly detection using a pre-trained ML model.
  • Interactive visualizations for insights into health event trends and anomalies.

How to Use

  1. Navigate to the project directory and build the pipeline using podman-compose build and podman-compose up.
  2. Verify that all services are running and healthy using podman ps.
  3. Access the live dashboard at http://localhost:5001.
  4. Query PostgreSQL data for insights into health events and anomaly predictions.

Troubleshooting

  • Ensure all containers are healthy by checking logs with podman logs.
  • If predictions are missing, verify the logs for the Predictor service.
  • If the visualization dashboard is inaccessible, confirm the Visualization container status and dependencies.
  • Check Kafka logs if it appears unhealthy or ports are blocked.
